## 2026-03-29 — Spacedock self-dogfood session

**Project:** Spacedock docs/plans workflow — managing design and implementation of Spacedock itself.
**Usage level:** Heavy. 8 active tasks, multiple concurrent agents (up to 3 parallel), worktree isolation, PR lifecycle via pr-merge mod.

### Friction: Agent cwd drift into worktrees

When the first officer uses `cd` to inspect a worktree (e.g., checking a rebase), the Bash cwd persists. Subsequent `Agent()` calls inherit that cwd, causing non-worktree-stage agents to resolve paths against the wrong directory. The 033 ideation ensign wrote its entire output into the 071 worktree instead of main. Content had to be manually copied over and the 071 branch reverted.

**Root cause:** FO template had no guidance on cwd management. Filed as task 072, fix is a "Working Directory" section in the FO template prohibiting `cd`.

### Friction: CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=0 has no effect

Setting the env var to 0 did not disable team tools. TeamCreate, SendMessage, TeamDelete were all still available in the tool list. The env var appears to be a feature flag for enabling teams (when set to 1), but setting it to 0 does not remove the tools from an already-enabled session. No known mechanism to disable team tools at runtime.

### Friction: Custom agent types frozen at session init

When the commission generates `.claude/agents/ensign.md` during Phase 2, the ensign agent type is NOT available for `Agent(subagent_type="ensign")` in Phase 3 of the same session. Agent types are loaded at session startup and not refreshed. The model recovered by falling back to `general-purpose` with inline instructions, but this means every commission auto-run hits this error silently.

### Friction: Model self-censoring on experiment design

The 033 experiment commissioned a workflow with mission "Test graceful degradation without agent teams." The model read the mission semantics and intentionally skipped calling TeamCreate — not because it was unavailable, but because the mission said "without agent teams." This confounded the experiment. Lesson: use neutral mission descriptions when testing failure modes.

### Friction: Validation rejected experiment scope

The validator correctly identified that the experiment only tested the commission's inline pilot (same session), not a fresh `claude --agent first-officer` session where generated agent files already exist on disk. The two-phase test (commission first, then fresh FO session) was never run. The experiment produced valuable findings (agent-type freezing) but didn't test the originally intended scenario.
